# WSL Ubuntu Ansible 基盤再設計計画

> **⚠️ 本書は再設計の背景計画です。現行仕様の正本ではありません。**
> 現行の仕様・運用・実装は [`docs/wsl-ubuntu-ansible/`](../../../../docs/wsl-ubuntu-ansible/) を正とし、本書は「今後長く使う前提でどのように作り直すか」を定義するための設計メモとして扱います。

## 1. 目的

この計画の目的は、いま動いている WSL Ubuntu Ansible を単に整理することではなく、次の条件に耐える **長期運用向けの土台** へ作り直すことです。

- role やツール追加のたびに playbook と変数参照が崩れない
- テスト導線を増やしても本体構成が濁らない
- `WSL 実機` と `Molecule 一時環境` の差分が構造として表現される
- 実行入口、設定、role、検証、docs の責務が混ざらない
- 今後 `cloud` や `security` を広げても、壊れ方が局所化される

## 1.1 今回の実装ゴール

この再設計計画は長期計画だが、今回のコーディングでの **明確な完了条件** は次に固定する。

- `playbooks/main.yml` と `playbooks/check.yml` を主入口として実装済みである
- 旧 `site*.yml` が `playbooks/compat/` に退避され、互換導線として残っている
- `group_vars/all.yml` の一枚依存が緩和され、少なくとも `profiles` / `platform` / `domains` の層が導入済みである
- `Makefile` と bootstrap 導線が新入口と variable layering を利用できる
- 基準テストとして `syntax` と `Molecule default/base/user-bootstrap` が通る

つまり、**Phase 1 と Phase 2 を実装完了にすること** を今回のゴールとする。

## 3. 未処理だけ

今後の実作業は次の 3 本に絞る。

### 3.1 role 再編

- `development-runtime` は当面、複合 role のまま `install_*_enabled` を持つ interface で維持する
- 将来分割するなら、この interface を保ったまま `runtime_*` role へ移す
- role ごとに `install` / `configure` / `verify` の責務をさらに揃える
- role README に入力変数、外部依存、verify 方法、Molecule 対応状況を揃える

### 3.2 テスト固定

- `runtime` / `postgresql` / `docker` の Molecule scenario を green 固定する
- `host-check` と role verify の責務差をこれ以上増やさない
- CI を入れるなら `syntax` / `lint` / `smoke` を最小ゲートにする

### 3.3 compat 縮退

- `playbooks/compat/` の実利用導線を確認する
- 非推奨注記をどこから入れるか決める
- `config.yml` 駆動の選択式 UI を profile へ寄せるかを判断する

## 4. 優先順位

次にやる順はこれで固定する。

1. `development-runtime` の分割方針を確定する
2. `runtime` Molecule を green 固定する
3. `postgresql` Molecule を green 固定する
4. `docker` Molecule を green 固定する
5. compat 縮退計画を作る

## 5. 破綻条件

この先で避けるべきことだけを残す。

- role 分割方針を決める前に、場当たりで task を足し続ける
- Molecule と実機差分を role 内分岐で増やし続ける
- profile を使わずに新しい playbook を増やす
- 正本 docs を更新せず、`ansible/docs/` 側だけで設計を進める
- compat を消すだけ先行し、既存運用の置き換え条件を決めない

## 6. 完了条件

この計画を完全完了と見なす条件は次。

- `development-runtime` の interface 方針が固定されている
- `runtime` / `postgresql` / `docker` の Molecule が安定 green
- role interface の説明が主要 role で揃っている
- compat 導線の残置理由か縮退計画が明文化されている

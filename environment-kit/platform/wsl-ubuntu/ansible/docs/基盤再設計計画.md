# WSL Ubuntu Ansible 基盤再設計計画

> **⚠️ 本書は再設計の背景計画です。現行仕様の正本ではありません。**
> 現行の仕様・運用・実装は [`docs/wsl-ubuntu-ansible/`](../../../../docs/wsl-ubuntu-ansible/) を正とし、本書は「今後長く使う前提でどのように作り直すか」を定義するための設計メモとして扱います。

## 1. 目的

この文書は、再設計そのものを説明するためではなく、**今回の再設計で残っている未処理だけ** を管理するためのメモである。

- 正本 docs に書くべき内容は `docs/wsl-ubuntu-ansible/` へ寄せる
- ここでは「何が終わり、何がまだ終わっていないか」だけを管理する
- 設計論や背景説明は増やさない
- 現行構成の説明は `01_仕様と設計.md` と `03_実装カタログ.md` を正とする

## 2. 現在地

再設計の土台はできており、現時点で見るべき差分は次だけである。

| 領域 | 状態 | 補足 |
|---|---|---|
| 主入口 | ✅ 完了 | `playbooks/main.yml` / `playbooks/check.yml` |
| 互換導線 | ✅ 完了 | `playbooks/compat/` に旧 `site*.yml` を隔離 |
| domain 導線 | ✅ 完了 | `playbooks/converge/` を導入 |
| 変数レイヤ | ✅ 完了 | `profiles` / `platform` / `domains` を導入 |
| Makefile / bootstrap | ✅ 完了 | 新入口と layered vars に追従 |

| verify / Molecule 固定 | ✅ 完了 | `runtime` / `postgresql` / `docker` を含む重い scenario まで green 確認済み |
| compat 縮退 | ✅ 方針固定 | `compat/README.md` と各 shim で非推奨方針を明示 |

## 3. 未処理だけ

今回の再設計計画で追っていた未処理は解消した。

- Molecule の重い scenario は `runtime` / `postgresql` / `docker` まで green
- role verify と scenario verify の責務差も現状で収束
- 次に増やすなら新規改善メモとして別文書で扱う

## 4. 優先順位

追加作業が発生した場合は、既存導線を壊さない小さな変更から扱う。

## 5. 破綻条件

この先で避けるべきことだけを残す。

- role 分割方針を決める前に、場当たりで task を足し続ける
- Molecule と実機差分を role 内分岐で増やし続ける
- profile を使わずに新しい playbook を増やす
- 正本 docs を更新せず、`ansible/docs/` 側だけで設計を進める

## 6. 完了条件

この計画を完全完了と見なす条件は次。

- `runtime` / `postgresql` / `docker` の Molecule が安定 green
- role interface の説明が主要 role で揃っている
- compat 導線の残置理由と非推奨方針が明文化されている

# Ansible 環境仕様・設計書（旧設計アーカイブ）

## 目的

* 個人ローカル（WSL 上）で開発環境を自動構築するための **最小・統一化 Ansible 設定**。
* **バージョンを `ansible.cfg` に集約**し、シングルソース管理で再現性を確保。
* **Playbook/Role 側はすべて `lookup('config', 'xxx_version')` で参照**する設計。

---

## 設定ファイル構成

```
ansible/
├─ ansible.cfg         # 本仕様に基づく統一設定
├─ inventory/
│   └─ hosts           # ローカルホスト定義
├─ site.yml            # 入口Playbook
├─ roles/              # 構成別Role群
│   ├─ common/         # 基本パッケージ
│   ├─ runtimes/       # Node/Go/Java/Maven/Gradle
│   ├─ containers/     # Docker/K8s関連
│   ├─ cloud/          # AWSCLI / Terraform / GitHub CLI
│   └─ database/       # PostgreSQL設定
└─ .vault_pass         # ansible-vault パスワード（ローカル専用）
```

---

## ansible.cfg の主要設定

### 基本設定

* `inventory = inventory/hosts`
* `connection = local`（WSL ローカル専用）
* `vault_password_file = .vault_pass`
* `log_path = /home/wsl/.ansible/ansible.log`

### 権限昇格

* `become = True`
* sudo 権限を自動利用

### ローカル作業用パス

* `download_dir = /home/wsl/.tmp-ansible`
* `local_tmp = /home/wsl/.tmp-ansible`
* `remote_tmp = /home/wsl/.tmp-ansible`

## 運用フロー

1. **バージョン変更**は `ansible.cfg` のみ修正すれば良い。
2. **Playbook/Role** 側は `lookup('config','nodejs_version')` のように参照。
3. **vault 管理**された sudo パスワードにより、毎回入力不要で `become: True` が利用可能。
4. **ログ**は `/home/wsl/.ansible/ansible.log` に追記される。
5. **一時ファイル**は `/home/wsl/.tmp-ansible` に保存され、tar.gz/zip 再利用も可能。

---

## 設計の特徴

* **完全ローカル専用**：WSL 内のみに閉じた設計。
* **シンプル化**：Ansible のデフォルトを極力利用、必要最低限の項目のみ記述。
* **再現性確保**：全ツール・ライブラリのバージョン固定。
* **セキュリティ簡易化**：ansible-vault に sudo パスワードを暗号化して保存、.vault\_pass で自動化。

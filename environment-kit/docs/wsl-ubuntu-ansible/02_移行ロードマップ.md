# 02 移行ロードマップ

構成の進化の経緯と、改善提案の現状（実装済 / 一部 / 未着手）を整理する。本書が改善ロードマップの正であり、旧 `wsl-ubuntu-ansible-improvement-plan.md` の要点もここに吸収した。詳細な背景メモとしては [`ansible/docs/refactoring-plan.md`](../../platform/wsl-ubuntu/ansible/docs/refactoring-plan.md) / [`ansible/docs/improvement-proposals.md`](../../platform/wsl-ubuntu/ansible/docs/improvement-proposals.md) / [`ansible/docs/基盤再設計計画.md`](../../platform/wsl-ubuntu/ansible/docs/%E5%9F%BA%E7%9B%A4%E5%86%8D%E8%A8%AD%E8%A8%88%E8%A8%88%E7%94%BB.md) を参照する。なお、`ansible/docs/` は現行仕様の正本ではなく、背景資料・旧設計アーカイブとして扱う。

## 1. これまでの進化

初期は単一 playbook（`site.yml` 相当）+ スクリプトのワンショット構成だった。現在は次まで進んでいる。

- **主入口の再定義**: `playbooks/main.yml` / `playbooks/check.yml` を主入口とし、旧 `site*.yml` は `playbooks/compat/` へ退避。
- **layered 変数化**: `group_vars/all.yml` をベースに `group_vars/profiles/` / `group_vars/platform/` / `group_vars/domains/` を導入。
- **role 分割**: `development-runtime` をサブタスク（rust/nodejs/golang/java/kotlin）に分解。`base` / `gcp` / `user-bootstrap` / `cache` を新設。
- **実行導線の再編**: `converge/` に domain 単位の導線、`compat/` に既存運用向け互換導線を分離。
- **検証の組み込み**: `playbooks/check.yml`、各 role の `validate.yml`、`script/check-system.sh`、Molecule。
- **言語/クラウド方針の確定**: ML=Python / 非 ML=Rust、メインクラウド=GCP、Java/Kotlin は opt-in。

## 1.1 Molecule シナリオ（8 種）

[`ansible/molecule/`](../../platform/wsl-ubuntu/ansible/molecule/) 配下に、Docker 一時環境で role / playbook を検証する Molecule シナリオを持つ。

| scenario | 主対象 | 目的 |
|---|---|---|
| `default` | smoke | Molecule 実行系の最小 smoke test |
| `base` | `cache` + `base` | WSL 基盤 role の導入と基本コマンド確認 |
| `user-bootstrap` | `user-bootstrap` | Git 設定、`.gitignore_global`、`.bashrc`、`.ssh` の idempotence 確認 |
| `runtime` | `development-runtime` | Rust / Node.js / Go 導入確認 |
| `cloud` | `aws` + `gcp` | AWS CLI / Terraform / gh / gcloud 導入確認 |
| `kubernetes-security` | `kubernetes` + `security` | 最小 Kubernetes ツールとセキュリティ CLI 確認 |
| `docker` | `docker` | Docker repo / CLI / compose plugin 確認 |
| `postgresql` | `postgresql` | PostgreSQL client / repo / neonctl 確認 |

Molecule では Docker コンテナを使うため、`timezone` / `locale` / `systemd service` のようなホスト依存処理は `platform/molecule` 変数で抑制する。`default` / `base` / `user-bootstrap` は基準 scenario、`runtime` / `postgresql` / `docker` は重い scenario として green 確認済みである。実ホストの最終確認は `playbooks/main.yml` / `playbooks/check.yml` 実行を正とする。

## 2. role（10 種）

| role | 導入/設定するもの | 主なゲート変数 |
|---|---|---|
| [`base`](../../platform/wsl-ubuntu/ansible/roles/base/) | 基本パッケージ、Python3 / pipx、**uv**、locale、timezone、`/etc/wsl.conf`、Doppler CLI | `python_uv_enabled`（既定 true） |
| [`user-bootstrap`](../../platform/wsl-ubuntu/ansible/roles/user-bootstrap/) | Git 設定、`~/.gitignore_global`、`~/.bashrc`（PATH/alias）、`~/.ssh` 権限 | `user_bootstrap_enabled`（既定 false） |
| [`development-runtime`](../../platform/wsl-ubuntu/ansible/roles/development-runtime/) | Rust（`/opt/rust`）、Node.js、Go（`/usr/local/go`）、(opt) Java+Maven+Gradle、(opt) Kotlin | `install_rust_enabled` / `install_nodejs_enabled` / `install_go_enabled` / `install_java_enabled` / `install_kotlin_enabled` |
| [`docker`](../../platform/wsl-ubuntu/ansible/roles/docker/) | Docker CE + Compose plugin、docker グループ追加 | `docker_version`（既定 system） |
| [`kubernetes`](../../platform/wsl-ubuntu/ansible/roles/kubernetes/) | 既定は kubectl/kind/kustomize/helm の最小セット。`kubernetes_install_*` で kubeadm/kubelet/minikube/containerd/crictl/nerdctl/krew/kubectx/kubens/k9s/stern を opt-in | `kubernetes_install_*` / `bash_completion` |
| [`security`](../../platform/wsl-ubuntu/ansible/roles/security/) | trivy/kube-bench/kube-hunter/kubescape、auditd、apparmor-utils | `auditd` / `apparmor_utils` |
| [`aws`](../../platform/wsl-ubuntu/ansible/roles/aws/) | AWS CLI v2、Terraform、GitHub CLI | — |
| [`gcp`](../../platform/wsl-ubuntu/ansible/roles/gcp/) | Google Cloud CLI（`/opt/google-cloud-sdk`） | `gcloud_version`（既定 latest） |
| [`postgresql`](../../platform/wsl-ubuntu/ansible/roles/postgresql/) | PostgreSQL 15 + contrib + PostGIS、DB（local_dev/local_test）、拡張、Neon CLI（neonctl, npm) | `postgres_*` |
| [`cache`](../../platform/wsl-ubuntu/ansible/roles/cache/) | ダウンロード/ビルドキャッシュ、APT キャッシュ、cron クリーンアップ、キャッシュ統計スクリプト | `preload_cache`（既定 true） |

> `user-bootstrap` は `env/config/setting.yaml` の `git_user_name` / `git_user_email` / `github_default_branch` を優先して Git グローバル設定へ反映する。
>
> 主要 role には `defaults/main.yml` と `README.md` を追加済みで、入力変数と責務を role 単位で読めるようにしている。

# Rust Batch ― 実装状況サマリ

> **対象ユースケース**: バッチジョブ実行・ワークフローランナー
> CLIまたはcronによるジョブスケジューリングを想定

---

## 技術スタック

| カテゴリ | 技術 |
|---------|------|
| Language | Rust (Edition 2021) |
| Runtime | Tokio (async) |
| Database | SQLite (rusqlite, bundled) |
| CLI | Clap v4 |
| Logging | Tracing + tracing-subscriber |
| Serialization | Serde, serde_yaml, serde_json |
| Date/Time | Chrono |
| Error Handling | thiserror, anyhow |
| ID Generation | UUID v4 |

## 環境設定 (env/config.yaml)

### 実装状態

| 項目 | 状態 |
|------|------|
| YAML設定ファイル読み込み | 実装済み (`env/config.yaml`) |
| 環境変数オーバーライド | 実装済み |
| 優先順位: 環境変数 > ファイル | 実装済み |
| デフォルト値フォールバック | 実装済み |

### 設定項目

| キー | 型 | 環境変数 | 備考 |
|------|-----|--------|------|
| projectName | string | `PROJECT_NAME` | デフォルト: 'batch' |
| appEnv | enum | `APP_ENV` | local/development/staging/production/test |
| runnerMode | enum | `RUNNER_MODE` | workflow/cli |
| logLevel | enum | `LOG_LEVEL` | trace/debug/info/warn/error |
| dbPath | string | `DB_PATH` | デフォルト: 'data/batch.db' |
| databaseUrl | string | `DATABASE_URL` | 外部DB接続用（Optional） |
| apiKey | string | `API_KEY` | 外部API呼び出し用（Optional） |

### 読み込み優先順位

1. 環境変数
2. 設定ファイル (`env/config.yaml`)
3. デフォルト値

### env/config.yaml

```yaml
# Application Configuration

projectName: batch
appEnv: local
logLevel: debug
dbPath: data/batch.db

# Runner mode: workflow | cli
runnerMode: workflow
```

## CLI使用方法

### コマンド

```bash
# ワークフロー実行（全ジョブ順次実行）
cargo run

# 特定ジョブの実行
cargo run -- --job hello-world
cargo run -- --job db-init
cargo run -- --job item-crud

# ジョブ一覧表示
cargo run -- --list

# ヘルプ
cargo run -- --help
```

### Makeターゲット

| ターゲット | 説明 |
|-----------|------|
| `make build` | デバッグビルド |
| `make release` | リリースビルド |
| `make run` | ワークフロー実行 |
| `make test` | テスト実行 |
| `make fmt` | コードフォーマット |
| `make clippy` | Linter実行 |
| `make hello-world` | hello-worldジョブ実行 |
| `make db-init` | db-initジョブ実行 |
| `make item-crud` | item-crudジョブ実行 |
| `make list` | ジョブ一覧表示 |

## Shared層 (src/shared/)

レイヤーに依存しない共通ユーティリティ。

| ファイル | 内容 |
|----------|------|
| `constants.rs` | EXIT_CODES, TIMEOUTS, LogLevel, AppEnv, RunnerMode |
| `utils.rs` | Timer, sleep, sleep_ms, mask_connection_string |

### 定数

| 定数 | 値 | 説明 |
|------|-----|------|
| `exit_codes::SUCCESS` | 0 | 正常終了 |
| `exit_codes::FAILURE` | 1 | 失敗 |
| `exit_codes::CONFIGURATION_ERROR` | 2 | 設定エラー |
| `timeouts::DEFAULT_JOB_MS` | 300,000 | デフォルトジョブタイムアウト（5分） |
| `timeouts::DB_CONNECT_MS` | 10,000 | DB接続タイムアウト |

## Interfaces層 (src/interfaces/)

トレイト定義（ポート）。テスタビリティのため抽象化。

### Clock

| メソッド | 戻り値 | 説明 |
|----------|--------|------|
| `now()` | i64 | エポックからのミリ秒 |
| `to_iso_string()` | String | ISO 8601形式の現在時刻 |
| `now_utc()` | DateTime<Utc> | UTC DateTime |

### ExecutionIdGenerator

| メソッド | 戻り値 | 説明 |
|----------|--------|------|
| `generate()` | String | ユニークな実行ID（`exec_{timestamp}_{uuid}`形式） |

### ConfigProvider

| メソッド | 戻り値 | 説明 |
|----------|--------|------|
| `get()` | &AppConfig | 現在の設定を取得 |

## Application層 (src/application/)

### ジョブタイプ

| 型 | 説明 |
|----|------|
| JobResult | ジョブ実行結果（success, message, executed_at, duration_ms, metrics, details） |
| PartialJobResult | タイミング情報なしの結果（ジョブ実装が返す） |
| JobContext | ジョブコンテキスト（job_name, execution_id, start_time） |
| JobError | ジョブエラー型（ExecutionFailed, ConfigError, DatabaseError等） |

### ジョブファクトリ

| 関数 | 説明 |
|------|------|
| `create_job(options, executor)` | 標準ジョブを作成（タイミング・ロギング自動化） |

### 登録済みジョブ

| ジョブ名 | ファイル | 説明 |
|---------|----------|------|
| hello-world | `jobs/hello_world.rs` | シンプルなHello Worldジョブ |
| db-init | `jobs/db_init.rs` | データベース初期化・マイグレーション実行 |
| item-crud | `jobs/item_crud.rs` | CRUD操作デモ（Create, Read, Update, Delete） |

## Registry層 (src/registry/)

グローバルジョブレジストリ。

| 関数 | 説明 |
|------|------|
| `register_job(job)` | ジョブを登録（重複時はpanic） |
| `get_job(name)` | 名前でジョブ取得 |
| `get_all_jobs()` | 全登録ジョブ取得 |
| `get_job_names()` | 全ジョブ名取得 |
| `has_job(name)` | ジョブ存在チェック |
| `clear_registry()` | レジストリクリア（テスト用） |

## データベースマイグレーション

### マイグレーションファイル

| ファイル | 説明 |
|----------|------|
| `db/migrations/001_init.sql` | 初期スキーマ（migrations, items テーブル） |

### スキーマ

```sql
-- マイグレーション追跡テーブル
CREATE TABLE migrations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  applied_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Itemsテーブル
CREATE TABLE items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

CREATE INDEX idx_items_created_at ON items(created_at DESC);
```

### マイグレーション実行

```bash
make db-init
# または
cargo run -- --job db-init
```

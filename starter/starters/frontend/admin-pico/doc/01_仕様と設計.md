# 01. 仕様と設計

本ドキュメントは、`pico-admin-template` の現行仕様を定義する。対象は **Pico CSS ベース / ダークテーマ固定 / デスクトップファースト** の静的管理画面テンプレートである。

---

## 1. 目的

このリポジトリの目的は、以下を満たす管理画面テンプレートを提供すること。

- `Tailwind` に依存しない
- `Pico CSS` の標準要素を主軸にする
- ダークテーマを固定し、テーマ切替を持たない
- デスクトップ PC での業務操作を第一目的にする
- ただしレスポンシブは維持し、狭幅でも破綻しない

---

## 2. デザイン原則

### 2.1 一点結論

**普段は無彩色、必要な瞬間だけライムが効き、危険操作だけピンクが刺さる管理画面にする。**

### 2.2 色の役割

| 用途 | 方針 |
|---|---|
| 背景 / カード / テーブル | 黒系 / ダークグレー系で統一 |
| 通常テキスト | 白系とグレー系 |
| 操作 / 成功 / フォーカス | ライムグリーン |
| 警告 / 異常 / 危険 | ショッキングピンク |

### 2.3 避けること

- 白いカード面
- 白いテーブル面
- 常時派手な色
- light / dark の切替 UI
- utility class の大量付与

---

## 3. UI 方針

### 3.1 デスクトップファースト

- 主設計面は `1280px 以上`
- サイドバーはデスクトップで常時表示
- KPI、テーブル、検索、操作ボタンは横方向前提で設計
- モバイル用メニューは補助導線に留める

### 3.2 レスポンシブ

レスポンシブは必須だが、思想は `mobile-first` ではなく **desktop-first の段階的圧縮**。

- `1500px+`
  - 高密度の多カラム
- `1320px 以下`
  - 4 カラム KPI を 2 カラムへ圧縮
- `1024px 以下`
  - サイドバーをオフキャンバス化
- `768px 以下`
  - ヘッダー操作群を縦積み

---

## 4. 技術方針

### 4.1 CSS

- `Pico CSS` を CDN import で利用
- 追加 CSS は `admin-*` 系ファイルに限定
- 色、余白、境界線は CSS 変数で統制
- 余白とコンポーネント高は業務画面向けに圧縮する

### 4.2 HTML

- 標準要素を優先
  - `main`
  - `aside`
  - `nav`
  - `header`
  - `section`
  - `table`
  - `input`
  - `button`
- 管理画面固有の表現だけ `admin-*` クラスを使う

### 4.3 JavaScript

- 最小限の素の JavaScript のみ使う
- 役割は以下に限定
  - サイドバー開閉
  - ダーク固定初期化
  - Chart.js 初期化

  <aside class="admin-auth__aside">...</aside>

## 7. コンポーネント仕様

### 7.1 サイドバー

- `details` / `summary` を使ったシンプルなナビ
- 現在地のみライム
- 未移行ページへの導線は持たない

### 7.2 カード

- ヘッダー面は `--admin-bg-header`
- 本文面は `--admin-bg-sub`
- 白いカード面は禁止
- 不要に背を高くしない

### 7.3 テーブル

- テーブル全体をダーク面で塗る
- 見出し行も白にしない
- hover は `--admin-bg-hover`
- デスクトップ前提で `min-width` を持たせる
- pager は `admin-pager` で統一する

### 7.4 チャート

- 見出しは短く保つ
- チャート面積は業務判断に必要な最小サイズへ抑える
- 装飾的な大きさを優先しない

### 7.5 バッジ

- `admin-badge--neutral`
  - 通常情報
- `admin-badge--lime`
  - 成功 / 正常 / 実行可能
- `admin-badge--pink`
  - 警告 / 要確認 / 危険

## 10. 非対象

以下は本リポジトリの現行スコープ外。

- `Tailwind` 再導入
- テーマ切替機能
- SPA 化
- バックエンド統合
- 未定義ページの再追加
